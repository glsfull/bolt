# Supabase migrations

Production-схема будет основана на миграциях из `project/supabase/migrations`,
но перед применением требует ревизии:

- RLS для всех пользовательских медицинских таблиц;
- индексы по `user_id`, датам анализов и статусам задач;
- приватные buckets для исходных файлов и PDF-отчетов;
- аудит административных изменений;
- роли `user`, `doctor`, `admin`;
- удаление аккаунта и связанных медицинских данных.

Файл `001_production_schema_plan.sql` фиксирует первый переносимый каркас
production-схемы: очереди анализа и отчетов, приватные Storage buckets,
настройки OCR/AI-провайдеров, feature flags и аудит администратора.
